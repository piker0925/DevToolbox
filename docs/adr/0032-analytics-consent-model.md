# ADR-0032 GA4 쿠키 동의 모델 — 기본 허용 + 손쉬운 거부

## 상태

확정 (2026-07-21)

## 배경

`/privacy` 처리방침 페이지를 실제 구현과 대조 검증하는 과정에서, 이미 라이브 중인 소셜 로그인·회원 탈퇴 기능을 "아직 제공되지 않음/도입 예정"이라고 서술하는 오류를 발견했다(즉시 정정). 같은 검증에서 GA4가 `main.ts`의 `initAnalytics()`를 통해 동의 없이 무조건 로드되고 있었고, 방침 페이지는 GA4 수집 사실만 명시할 뿐 동의·거부 수단이 전혀 없다는 것도 함께 드러났다.

한국 개인정보보호법 체계는 EU GDPR/ePrivacy식 "쿠키 사용 전 사전 옵트인 동의"를 일반적으로 요구하지 않는다. GA4 같은 비필수(non-essential) 분석 쿠키는 방통위·KISA의 온라인 맞춤형광고 가이드라인이 요구하는 "고지 + 손쉬운 옵트아웃" 수준으로 충분하다고 판단했다.

## 결정

**기본 허용(default-allow) + 상시 노출 알림 배너 + 언제든 되돌릴 수 있는 옵트아웃**을 택한다. EU식 차단형 사전 동의(배너를 눌러야만 GA4가 로드되는 방식)는 한 차례 구현했다가, 실제로 필요한 수준보다 과한 마찰을 준다는 이유로 이 ADR의 모델로 대체했다.

- **`front/src/config/consent.ts`**: `localStorage`에 `granted`/`denied` 중 하나만 저장. 값이 없으면(미결정) "기본 허용"으로 취급 — `null`을 별도의 차단 상태로 쓰지 않는다.
- **`front/src/config/analytics.ts`**: `initAnalytics()`는 `getConsent() === 'denied'`일 때만 스크립트 주입을 건너뛴다(미결정·`granted` 둘 다 로드). 이미 주입된 스크립트가 있으면 재호출 시 중복 주입하지 않는다(idempotent) — 부팅 시 자동 로드와 배너의 "확인" 클릭이 겹칠 수 있어 필요해졌다. `disableAnalytics()`는 주입된 `<script>` 태그를 제거하고 `window.gtag`/`window.dataLayer`를 지워 이미 로드된 이후에도 거부 시 즉시 추적을 멈춘다(단, 거부 이전에 이미 발생한 GA 요청·설정된 쿠키 자체를 소급 취소하지는 못한다 — 이 한계는 수용).
- **`front/src/components/CookieConsentBanner.vue`**: `App.vue` 최상위에 배치해 bare 레이아웃(`/`)과 `DefaultLayout` 양쪽을 모두 커버. GA_ID가 설정돼 있고 아직 결정하지 않았을 때만 하단에 알림형 배너를 띄운다 — "동의를 구하는 차단막"이 아니라 "이미 쓰고 있다는 고지 + 거부 버튼"이다. 선택(확인/거부) 즉시 `localStorage`에 영구 저장되고 이후 재방문·새로고침에서 다시 뜨지 않는다.
- **`front/src/pages/PrivacyPage.vue`**: "쿠키 설정" 섹션에 현재 상태 표시 + 토글 버튼을 둬, 배너를 무시했거나 이미 결정한 사용자도 마음을 바꿀 수 있는 상시 경로를 제공한다.

## 결과

- GA4는 명시적 거부가 없는 한 항상 수집한다 — "사용자가 배너를 안 보고 넘어가도 무방하다"는 게 이 모델의 의도된 동작이다.
- 배너 문구에 "거부해도 서비스 이용엔 지장 없다"는 점을 명시해 거부가 손해로 이어지지 않음을 알린다.
- 검증: `consent.test.ts`(저장·조회 왕복, 오염값 방어) · `analytics.test.ts`(미결정 시 기본 로드, 거부 시 미로드, 중복 미주입, `disableAnalytics` 정리) · `CookieConsentBanner.test.ts`(노출 조건, 클릭 후 저장·소멸, 이미 로드된 상태에서 거부 시 실제 제거) · `PrivacyPage.test.ts`(상태별 토글 라벨·클릭 동작) — `pnpm test`(1193개) + `pnpm run build` 통과. 실브라우저로 임시 `VITE_GA_ID`를 심어 배너 노출 → 거부 시 `googletagmanager.com` 요청 없음 → 동의 시 실제 요청 발생(200) → 재거부 시 스크립트 태그 제거·`window.gtag` undefined까지 Network 탭·DOM에서 직접 확인.
- 처리방침 페이지의 소셜 로그인·회원 탈퇴 서술 오류도 같은 세션에서 함께 정정하고 개정 이력에 반영했다(별도 결정 사항 아님, 사실관계 정정).
